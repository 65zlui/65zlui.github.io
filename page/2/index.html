原神—缝合怪 崩崩崩+荒野之息


这是一部缝合怪作品，感觉就是把荒野之息的元素和崩坏三缝合，
崩坏三的元素：
如抽卡，多个角色之间的切换，打击和技能，感觉都是崩坏三的延续，
荒野之息的元素

开放世界，野炊，收集，画风，

这是抽卡游戏，也有开放世界，有很多元素的混合，但是它们糅合后的体验并不算佳，
画面在手游中完全可以算一线水准，
如果把它当作pc网游其实也还说得过去，不过玩家间互相交互的元素稍微少了点，16级前原生只是有抽卡机制的单机。
游戏机制，在很多方面让你感觉米哈游是想引导你花更多的时间肝主线，
原神前20-30个小时的体验，其实相当不错，

到了一定等级后，升级越来越难，演变成网游常见的刷boss肝任务，开放世界就成了一个谎言，原神只是有着开放世界的皮，
原神之后的体验就渐渐变得无聊起来，
比如冒险等级机制，不让你速通主线，逼你花更多的时间让你肝在游戏里
剧情和巫师三那种不能比，叙事非常单薄低幼。
如果把原神当作一个抽卡网游，其实它是算相对优秀的作品
如果把它和塞尔达比较，没有太多可比性，一个内购氪金游戏和一个买断制游戏感觉比较起来没那么大可比性。

未完待续。。。

title: OS-1-4 分时操作系统
date: 2020-09-08 22:30:52
tags:

多道批处理系统的缺点
（平均周转时间长）作业处理时间长；
交互能力差；
运行过程不确定。
不用不一定是空闲的，
共享，用忙，同时等cpu的进程可能有很多，
宏观上并行，

  • 事务性任务的涌现
    • 交互度高
    • 相应快速
  • 要求支持多任务/多用户
  • 多终端计算机
    • 高性能主机+多个终端
      • 主机：运算，cpu + 内存
      • 终端：输入和显示
    • 主机采用分时技术轮流为每个终端服务，每个终端都感觉到
  • 分时技术
    • 概念
      • 主机以很短的时间片为单位，把cpu轮流分配给终端使用，直到全部作业被运行完
      • 由于时间片很短，在终端数量不多的情况下，每个终端都能很快重新获得cpu，使得每个终端都能得到及时响应
        • 等待周期 = 时间片 * 终端数量
    • 特点
      • 多路调制性
        • 多用户联机使用同一台计算机
      • 独占性
        • 用户感觉独占计算机
      • 交互性
        • 及时响应用户的请求
  • 大型分时系统的实践：multics项目
    • 1962年由apra支持，有mit、bell和G.E参与
    • 开发一种“公用计算服务系统”
    • multiplexed information and computing service
    • 同时支持波士顿地区的用户
  • unix
    • 第一个实用化的分时操作系统
      • 第一个真正体现操作系统领域各种先进概念和技术的操作系统
    • 革新和创造
      • 实现操作系统的可移植性
      • 实现了硬件无关性
      • 引进了“特殊文件”（special file）的概念
        • 把外设看作文件，实现对外设统一管理

Mad Game Tycoon ——疯狂游戏大亨


以前一直想在电脑上游玩类似游戏发展国的游戏，开罗确实做过一个pc的游戏发展国，不过画面太过差了。
之后在逼乎上看到有人推荐mad game tycoon，我决定一试，耐玩性确实要比游戏发展国好些，员工的数量只要你想要，就可以拥有一个几百人的员工队伍，这更接近于现实中一个游戏公司的情况。
但是没有创业公司startup company这部游戏中的经理系统，当公司规模大起来后，去管理到一个员工个人是很麻烦的事情，如果能交由手下的经理管理就好多了。
画面确实不是很好，但总比开罗的pc版本游戏发展国要好太多了

  1. 优点：内容丰富，单论此游戏中可以建立的部门就有：客服中心，硬件开发部，技术研究部，营销中心，总办公室，员工培训室，生产车间，存储室，图像处理部，录音室，质量监控室。
  2. 缺点：薄弱的画面和音乐

整体评分 B+ 值得一玩
游戏性 A+
画面 C
