date: 2020-09-08 22:30:52
tags:

多道批处理系统的缺点
（平均周转时间长）作业处理时间长；
交互能力差；
运行过程不确定。
不用不一定是空闲的，
共享，用忙，同时等cpu的进程可能有很多，
宏观上并行，

  • 事务性任务的涌现
    • 交互度高
    • 相应快速
  • 要求支持多任务/多用户
  • 多终端计算机
    • 高性能主机+多个终端
      • 主机：运算，cpu + 内存
      • 终端：输入和显示
    • 主机采用分时技术轮流为每个终端服务，每个终端都感觉到
  • 分时技术
    • 概念
      • 主机以很短的时间片为单位，把cpu轮流分配给终端使用，直到全部作业被运行完
      • 由于时间片很短，在终端数量不多的情况下，每个终端都能很快重新获得cpu，使得每个终端都能得到及时响应
        • 等待周期 = 时间片 * 终端数量
    • 特点
      • 多路调制性
        • 多用户联机使用同一台计算机
      • 独占性
        • 用户感觉独占计算机
      • 交互性
        • 及时响应用户的请求
  • 大型分时系统的实践：multics项目
    • 1962年由apra支持，有mit、bell和G.E参与
    • 开发一种“公用计算服务系统”
    • multiplexed information and computing service
    • 同时支持波士顿地区的用户
  • unix
    • 第一个实用化的分时操作系统
      • 第一个真正体现操作系统领域各种先进概念和技术的操作系统
    • 革新和创造
      • 实现操作系统的可移植性
      • 实现了硬件无关性
      • 引进了“特殊文件”（special file）的概念
        • 把外设看作文件，实现对外设统一管理

  • 硬件发展的四个典型阶段

    • 电子管时代[1946-1955]
    • 晶体管时代[1955-1965]
    • 集成电路时代[1965-1980]
    • 大规模集成电路时代[1980-now]

  • 操作系统发展的四个典型阶段

    • 手工操作 (无操作系统) early 1950s
    • 单通批处理系统 1950s
    • 多道批处理系统 early 1960s
    • 分时系统 mid 1960s

  • 第一台电子计算机ENIAC

  • 电子管时代 1946-1955（手工操作）

    • 结构特点
      • 硬件：电子管，接线面板
      • 程序： 二进制程序，打孔，纸带和卡片
    • 使用特点
      • 程序准备/启动结束：手工处理，繁琐耗时
    • 缺点
      • 效率低
      • 用户独占
      • 缺少交互

  • 单道批处理系统（50s）

    • 管理员事先将多个作业输入到磁盘形成作业队列
    • 操作系统依次自动处理队列中的每个作业
      • 装入-运行-撤出
    • 程序的装入和撤出：自动完成，效率高；
    • 运行完毕，通知用户取结果

  • 工作特点

    • 批量：作业队列
    • 自动：识别作业，装入和撤出
    • 单道：

  • 多道批处理系统（60s初）

    • 多道：内存同时存放多道程序
    • 并行：宏观上
    • 串行：微观上

  • 分时系统（60年代中）

    • 时间片：按时间片轮转的方式工作

  • 实时操作系统（60年代中）

  1. 操作系统的定义：
    1. 提供接口；
    2. 分配和调度软件资源；

操作系统在计算机系统中处于中间层面，承上启下；

  1. 操作系统的功能
    1. 进程管理（cpu管理）
      1. 进程控制：创建，暂停，唤醒，撤销；
      2. 进程调度：调度策略，优先级；
      3. 进程间的通信；
    2. 内存管理
      1. 内存分配
      2. 内存管理
      3. 内存保护
      4. 虚拟内存
    3. 设备管理
      1. 设备间的分配和调度
      2. 设备无关性
      3. 设备传输机制
      4. 设备驱动
    4. 文件管理
      1. 储存空间管理
      2. 文件的操作
      3. 目录的操作
      4. 文件和目录的存储权限管理
    5. 网络管理
    6. 分布式管理功能
    7. 图形界面
